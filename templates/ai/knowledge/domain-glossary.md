# Domain Glossary

This file is L0.5 long-term project knowledge for stable domain vocabulary.

## Entry Format

- **Term**: [stable technical or product term]
- **Meaning**: [project-specific meaning]
- **Owner / Boundary**: [repository, module, API, or workflow boundary]
- **Evidence**: [source file, spec, commit, owner confirmation, or validation]

## Initial Terms

- **UUID decimal string**
  - **Meaning**: The only cross-boundary device identity exposed through Libs,
    Biz, N-API/JSON/RPC, JS, and UI.
  - **Owner / Boundary**: Generated by `device::identity::generateUUID()`;
    consumed outside the lowest SDK integration layer.
  - **Evidence**: Team constitution.

- **NativeBridge**
  - **Meaning**: API forwarding bridge between frontend plugin and downstream
    utility APIs.
  - **Owner / Boundary**: Must not own runtime truth, permission rules, or UI
    behavior inference.
  - **Evidence**: Team constitution.
