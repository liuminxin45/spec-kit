# Project Overview

This file is L0.5 long-term project knowledge. It should summarize durable
project facts, not per-feature decisions.

## Purpose

Record stable information that helps future AI agents orient quickly:

- product/domain context
- important repositories and ownership boundaries
- recurring build, package, or runtime constraints
- durable validation expectations

## Current Source Of Truth

- Repository map: `.specify/memory/repository-map.md`
- Workspace config: `.specify/workspace.yml`
- Confirmed pitfalls: `.specify/memory/pitfalls.md`
- Team principles: `.specify/memory/constitution.md`

## Governance

AI may propose updates to this file only when it has:

- source evidence
- reason for promotion from feature-local notes
- validation or owner-confirmed evidence

Normal feature workflows should write candidates to `workflow-record.md`,
`improvement-candidates.md`, or `promotion-report.md` before changing this
long-term knowledge file.
